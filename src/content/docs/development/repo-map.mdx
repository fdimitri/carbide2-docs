## Local checkout gotchas

### The worker symlink

`carbide2-server/worker` is a **symlink** to the sibling `carbide2-worker` checkout.
Editing `carbide2-server/worker/*.rb` edits the worker repo — commit those changes
in `carbide2-worker`, not `carbide2-server`.

### Client checkout

The real client is the sibling `carbide2-client` checkout;
`carbide2-server/clients/carbide2-client` is a partial copy. The
`CARBIDE2_CLIENT` env var overrides the default sibling path.

### Docs vs. public site

Edit `carbide2-docs` for architecture, API, ops, and contributor documentation.
Edit `carbide2-site` for the public website, landing pages, and shared brand
assets that may also be reused by the docs site.

### Shared database

The worker connects to the **same Postgres** as Rails (`worker/ar_boot.rb`) and
loads every `app/models/*.rb`. Schema changes affect both processes — see
[Database Schema](/architecture/database-schema/).
