This page defines the current chat-coupled video signaling contract carried over the worker WebSocket.

Status:
- Implemented but rudimentary.
- Coupled to chat room membership.
- Signaling is relay-only; media is peer-to-peer.

Security:
- Required level: N2 for all rtc commands.
- Caller must be a member of the workspace project and joined to the target room.

Envelope:
- Uses the standard WS shape with `cs`, `cmd`, and `payload`.
- `cs` value is always `rtc`.

Commands:

| Command | Direction | Payload | Notes |
|---|---|---|---|
| `join` | Câ†’S | `{ channel_id }` | Caller must already be joined to that chat channel |
| `peers` | Sâ†’C | `{ channel_id, peers }` | Sent to the joiner with current call peers |
| `peer_join` | Sâ†’C | `{ channel_id, peer_id, user_id, name }` | Broadcast to existing call members |
| `leave` | Câ†’S | `{ channel_id }` | Leaves the live call for that channel |
| `left` | Sâ†’C | `{ channel_id }` | Reply to the leaver |
| `peer_leave` | Sâ†’C | `{ channel_id, peer_id, user_id, name }` | Broadcast to remaining call members |
| `signal` | Câ†’S | `{ channel_id, to, data }` | Relay arbitrary SDP/ICE payload to a specific peer |
| `signal` | Sâ†’C | `{ channel_id, from, data }` | Forwarded unchanged except `from` is the sender peer id |
| `call_state` | Sâ†’C | `{ channel_id, participants }` | Presence snapshot sent to all channel members |

Semantics:
- Worker relays only. It does not parse or mutate SDP or ICE candidates.
- No persistence of rtc signaling frames.
- On reconnect, client must renegotiate.
- Full mesh is assumed for small rooms.

Error cases:
- unauthorized: caller lacks N2 or room membership.
- unknown_peer: target peer is not currently present in the call.
- room_mismatch: caller is not joined to the target channel call.
- malformed_payload: required fields missing or invalid.

Concrete example flow:

Caller joins the call for chat channel `1`:

```json
{ "cs": "rtc", "cmd": "join", "payload": { "channel_id": 1 } }
```

Worker replies with the peers the caller should offer to:

```json
{
  "cs": "rtc",
  "cmd": "peers",
  "payload": {
    "channel_id": 1,
    "peers": [
      { "peer_id": "b7", "user_id": 9, "name": "bob" }
    ]
  }
}
```

Caller sends a signaling blob to peer `b7`:

```json
{
  "cs": "rtc",
  "cmd": "signal",
  "payload": {
    "channel_id": 1,
    "to": "b7",
    "data": {
      "type": "offer",
      "sdp": "OFFER_SDP"
    }
  }
}
```

Worker forwards it to that peer as:

```json
{
  "cs": "rtc",
  "cmd": "signal",
  "payload": {
    "channel_id": 1,
    "from": "a1",
    "data": {
      "type": "offer",
      "sdp": "OFFER_SDP"
    }
  }
}
```

Compatibility notes:
- This contract is best effort and may evolve.
- Future change should version command payloads explicitly if breaking fields are introduced.
