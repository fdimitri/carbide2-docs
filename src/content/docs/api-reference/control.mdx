## Sessions

### `POST /api/login`

Log in — issue a control-plane session JWT.

**Request body:**
```json
{ "email": "user@example.com", "password": "..." }
```

**Response:**
```json
{ "token": "<jwt>", "user": { "id": 1, "email": "..." } }
```

### `POST /api/signup`

Register a new user.

### `DELETE /api/logout`

End the current session.

## Workspaces

### `GET /api/workspaces`

List the authenticated user's workspaces, including pod health.

### `POST /api/workspaces`

Create a new workspace. The k8s operator provisions a `ws-<id>` namespace and pod.

### `GET /api/workspaces/:id`

Workspace detail, including provisioning status and health.

### `DELETE /api/workspaces/:id`

Deprovision a workspace (operator tears down the namespace).

### `POST /api/workspaces/:id/token`

Mint a workspace-scoped JWT for connecting to the workspace pod's API/WebSocket.

### `GET /api/workspaces/:id/health`

Pod liveness/readiness summary for the dashboard health dot.

## Concrete examples

### Example: create workspace

```bash
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"my-workspace"}' \
  "http://localhost:3001/api/workspaces"
```

Example response:

```json
{
  "id": "ws_123",
  "name": "my-workspace",
  "status": "provisioning",
  "public_url": "https://example.local/w/ws_123/"
}
```

### Example: mint workspace token

```bash
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  "http://localhost:3001/api/workspaces/ws_123/token"
```
