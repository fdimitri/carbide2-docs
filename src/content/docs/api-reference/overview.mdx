CARB/IDE2 exposes two REST APIs and one WebSocket protocol:

| API | Base URL | Auth |
|-----|----------|------|
| **Server API** (`carbide2-server`) | `/api/` (per workspace) | JWT Bearer |
| **Control Plane API** (`carbide2-control`) | `/api/` (main domain) | JWT Bearer |
| **WebSocket protocol** (`carbide2-worker`) | `/w/<id>/ws` | JWT query param |

## Authentication

All REST endpoints require a valid JWT in the `Authorization` header:

```
Authorization: Bearer <token>
```

Tokens are issued by `POST /api/sessions` on the control plane. See
[Authentication & JWT](/architecture/authentication/) for the full token spec.

## Security levels

All endpoint and command tables now include a `Security` column using the model
defined in [Security Model](/architecture/security-model/):

- `N0` public
- `N1` authenticated user
- `N2` project/workspace member
- `N3` privileged or owner action

## Versioning

APIs are currently unversioned. Breaking changes will be communicated via this
documentation and the GitHub changelog.
