## Authentication

### `POST /api/login`

Exchange a **control-plane bearer token** for a workspace user token. This is
not an email/password login — the workspace pod delegates credential checks to
`carbide2-control` (see [Authentication & JWT](/architecture/authentication/)).

**Request:** `Authorization: Bearer <control-plane workspace token>`

**Response:**
```json
{ "token": "<workspace user jwt>", "user": { "id": 1, "email": "..." }, "message": "Login successful" }
```

### `POST /api/signup`

Always returns **405** — account creation is handled by the control plane
dashboard, not by workspace pods.

## Concrete examples

### Example: fetch the project tree

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:3000/api/projects/1/fs/tree"
```

The response is the full nested tree for the project:

```json
{
  "name": "/",
  "path": "/",
  "ftype": "folder",
  "children": [
    { "name": "src", "path": "/src", "ftype": "folder", "children": [] },
    { "name": "README.md", "path": "/README.md", "ftype": "file" }
  ]
}
```

### Example: create a file

```bash
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"path":"/src/new_file.rb","content":"puts :ok\n"}' \
  "http://localhost:3000/api/projects/1/fs/files"
```

## Projects

### `GET /api/projects`

List all projects the authenticated user is a member of.

### `POST /api/projects`

Create a new project.

**Request body:**
```json
{ "project": { "name": "my-project" } }
```

### `GET /api/projects/:id`

Get a single project.

### `PATCH /api/projects/:id`

Update project metadata.

### `DELETE /api/projects/:id`

Delete a project. Owner only.

### `POST /api/projects/:id/ws_token`

Mint a project-scoped worker JWT for the WebSocket connection (legacy path;
see [Authentication & JWT](/architecture/authentication/)).

### `GET` / `PATCH /api/projects/:id/settings`

Read or update per-project settings (DBFS flush tuning, `root_path`,
`shell_image`, …).

## Filesystem (DBFS)

All DBFS endpoints are scoped to a project: `/api/projects/:project_id/fs/...`

<Aside type="tip">
  The DBFS is also accessible over WebSocket via the `fs` commandSet.
  See [WebSocket Commands](/api-reference/websocket-commands/).
</Aside>

### `GET /api/projects/:project_id/fs/tree`

Return the full nested file tree for the project.

### `GET /api/projects/:project_id/fs/content`

Read a file's current content (replays the `FileChange` log via `calc_current`).

**Query params:** `path` (e.g. `?path=/src/app.rb`)

### `GET /api/projects/:project_id/fs/stat`

Stat-style metadata for a single entry. **Query params:** `path`

### `GET /api/projects/:project_id/fs/blob`

Raw bytes for a binary entry. **Query params:** `path`

### `POST /api/projects/:project_id/fs/files`

Create a new file.

**Request body:**
```json
{ "path": "/src/app.js", "content": "", "mkdirp": true }
```

### `POST /api/projects/:project_id/fs/dirs`

Create a directory (`mkdir -p` semantics).

**Request body:** `{ "path": "/src/lib" }`

### `PATCH /api/projects/:project_id/fs/rename`

Rename an entry.

**Request body:** `{ "path": "/src/old.rb", "new_name": "new.rb" }`

### `DELETE /api/projects/:project_id/fs/entry`

Delete a file or directory (and its children).

**Request body:** `{ "path": "/src/old.rb" }`

### `POST /api/projects/:project_id/fs/upload`

Upload an archive (zip, tar, tar.gz) or single file and import it into the DBFS.

**Content-Type:** `multipart/form-data`

### `POST /api/projects/:project_id/fs/import`

Import the project's on-disk root into the DBFS (used after out-of-band
filesystem population).

## Terminal recordings

### `GET /api/projects/:project_id/recordings`

List asciinema cast recordings for the project.

### `GET /api/projects/:project_id/recordings/:id/cast`

Download the asciinema v2 cast file for replay.

### `DELETE /api/projects/:project_id/recordings/:id`

Delete a recording.

## Preferences

### `GET /api/preferences`

Return the authenticated user's preferences (theme, editor font size, tab
width, timezone, …), creating a default row on first access.

### `PATCH /api/preferences`

Update the authenticated user's preferences.
