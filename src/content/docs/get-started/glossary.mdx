| Term | Meaning |
|------|---------|
| **DBFS / VFS** | The database-backed virtual filesystem. Edits persist continuously as `FileChange` rows — there is **no save step** in Carbide2. |
| **Workspace** | One provisioned environment = one Kubernetes pod = one project (Model B). Managed by the control plane. |
| **Project** | The workspace-local Rails record owning the file tree, chat, agents. Its primary key is local — unrelated to the control-plane workspace id. |
| **Control plane** | `carbide2-control` — dashboard, workspace CRUD, k8s operator. Serves `/` on the main domain. |
| **Workspace pod** | The per-workspace deployment (`ws-<id>` namespace) running Rails + worker. Serves `/w/<id>/`. |
| **Worker** | The EventMachine WebSocket server (`carbide2-worker`) — terminals, chat, fs, agents. |
| **commandSet (`cs`)** | Top-level routing key in WS messages: `system`, `fs`, `terminal`, `chat`, `agents`. |
| **`calc_current`** | `DirectoryEntry` method that replays the `FileChange` log to produce current file content. |
| **VfsFlusher** | Worker background loop that writes DBFS content to pod disk (timer + byte threshold). |
| **Meta repo** | `carbide2` — pins the four component repos as submodules; owns `deploy.rb`. |
| **Agent** | An LLM persona (model + system prompt + tool allowlist) chat-able from the workspace. |
| **Cast / recording** | asciinema v2 file capturing terminal output (`terminal_recordings`). |
